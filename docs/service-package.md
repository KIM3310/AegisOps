# Service Package - AegisOps

This package defines the most practical service shape for this repository. It is public-safe: no price promises, financial projections, or buyer-specific contract assumptions are published here.

## Buyer and Wedge

- **Service lane:** Security and operations cockpit
- **Primary buyer:** SOC, NOC, MSP, and infrastructure teams with noisy handoff loops
- **Front-door offer:** Incident replay and shift-handoff tabletop
- **Service outcome:** Turn a synthetic incident into a reusable evidence pack, handoff board, and executive-ready report.

## Build and Polish Priorities

- Tighten the first-screen incident summary and next-action hierarchy
- Add export-ready runbook/report states for buyer review
- Keep demo data synthetic while making escalation paths realistic

## Buyer-Ready Deliverables

- Scenario pack
- Handoff board
- Runbook export
- Monthly review cadence

## Activation Path

1. Use the public demo, README, screenshots, and tests to establish proof.
2. Confirm the buyer workflow, data boundary, owner, and acceptance criteria.
3. Build only the smallest workflow needed to prove the service outcome.
4. Add cloud resources, accounts, storage, or paid integrations only after the workflow boundary is approved.

## Margin and Risk Controls

- Keep support bounded with reusable scenarios, report templates, and a fixed escalation boundary.
- Keep the first service scope narrow, repeatable, and evidence-led.
- Keep private financial planning outside the public repository.
- Keep logs, analytics, and reports free of private payloads unless explicit approval exists.

## Portfolio Signal

The portfolio should present this repository as **Incident replay and shift-handoff tabletop** under the **Security and operations cockpit** lane, with proof links routed to the live demo or repository evidence.
